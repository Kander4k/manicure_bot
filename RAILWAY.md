# Пошаговая инструкция: деплой бота на Railway

Следуйте шагам по порядку. В конце бот будет работать 24/7 в облаке.

---

## Шаг 1. Залить проект на GitHub

Если репозитория ещё нет:

1. Зайдите на [github.com](https://github.com) и войдите в аккаунт.
2. Нажмите **New repository** (или «+» → New repository).
3. Название, например: `manicure_bot`. Репозиторий может быть **приватным**.
4. Нажмите **Create repository**.
5. На компьютере откройте терминал в папке проекта `manicure_bot` и выполните:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/manicure_bot.git
git push -u origin main
```

Подставьте вместо `ВАШ_ЛОГИН` свой логин GitHub. Если Git спрашивает логин/пароль — используйте **Personal Access Token** (Settings → Developer settings → Personal access tokens).

**Важно:** в репозиторий не должен попасть файл `.env` (токены и пароли). Добавьте в корень проекта файл `.gitignore` с содержимым:

```
.env
venv/
__pycache__/
*.pyc
*.db
```

Если `.gitignore` уже есть — убедитесь, что в нём есть строка `.env`. После этого снова:

```bash
git add .
git commit -m "Add .gitignore"
git push
```

---

## Шаг 2. Зарегистрироваться на Railway

1. Откройте [railway.app](https://railway.app).
2. Нажмите **Login** → **Login with GitHub**.
3. Разрешите Railway доступ к вашему GitHub-аккаунту (при необходимости выберите только нужный репозиторий).

---

## Шаг 3. Создать проект и подключить репозиторий

1. В Railway нажмите **New Project**.
2. Выберите **Deploy from GitHub repo**.
3. Если репозиторий не виден — нажмите **Configure GitHub App** и разрешите доступ к репозиторию `manicure_bot`.
4. Выберите репозиторий **manicure_bot** и нажмите на него.
5. Railway создаст сервис и начнёт первую сборку. Пока что она может завершиться с ошибкой — это нормально, дальше настроим команду запуска и переменные.

---

## Шаг 4. Указать команду запуска

1. В Railway откройте созданный сервис (клик по сервису).
2. Перейдите на вкладку **Settings** (или **Variables** → рядом есть настройки).
3. Найдите блок **Deploy** или **Build & Deploy**.
4. В поле **Start Command** (или **Custom Start Command**) введите:

   ```
   python bot.py
   ```

5. Сохраните (если есть кнопка **Save** или настройки сохраняются автоматически).

Если в интерфейсе есть **Root Directory** — оставьте пустым или укажите `/` (корень репозитория).

---

## Шаг 5. Добавить переменные окружения

Без них бот не запустится. Переменные берём из вашего локального файла `.env`.

1. В том же сервисе откройте вкладку **Variables** (или **Environment**).
2. Нажмите **New Variable** или **Add Variable** и добавьте по одной переменной:

| Имя переменной | Значение (пример) | Откуда взять |
|----------------|-------------------|--------------|
| `BOT_TOKEN` | `7123456789:AAH...` | Токен бота от @BotFather в Telegram |
| `ADMIN_ID` | `123456789` | Ваш Telegram user ID (число) |
| `CHANNEL_ID` | `-1001234567890` | ID канала (число, с минусом для каналов) |
| `CHANNEL_LINK` | `https://t.me/your_channel` | Ссылка на канал |

Значения скопируйте из вашего `.env` на компьютере. **ADMIN_ID** и **CHANNEL_ID** — только числа, без кавычек.

3. После добавления всех переменных Railway обычно перезапускает деплой сам. Если нет — перейдите на вкладку **Deployments** и нажмите **Redeploy** (три точки у последнего деплоя → Redeploy).

---

## Шаг 6. Проверить деплой

1. На вкладке **Deployments** дождитесь статуса **Success** (зелёная галочка).
2. Откройте вкладку **Logs** (или **View Logs**). В логах должно появиться сообщение вроде: **«Бот запущен...»** — значит, бот стартовал.
3. В Telegram откройте вашего бота и отправьте команду `/start`. Бот должен ответить.

Если в логах ошибка — см. раздел «Частые ошибки» ниже.

---

## Шаг 7. (По желанию) Сохранить базу между перезапусками

По умолчанию база SQLite лежит в файле внутри контейнера. При новом деплое или перезапуске файл может сбрасываться.

- Если пока устраивает, что данные могут обнуляться при редеплое — ничего делать не нужно.
- Если нужна постоянная база:
  - В Railway в проекте можно добавить **Volume** (том) и примонтировать его к папке, где создаётся `database.db`, либо
  - Позже перейти на PostgreSQL в Railway и поменять код бота на работу с PostgreSQL.

Для первого запуска можно оставить как есть и при необходимости настроить том позже.

## Частые ошибки

**В логах: «Не указан BOT_TOKEN» (или другая переменная)**  
→ Вернитесь в **Variables** и проверьте, что все четыре переменные заданы без опечаток в именах: `BOT_TOKEN`, `ADMIN_ID`, `CHANNEL_ID`, `CHANNEL_LINK`.

**В логах ошибка сборки (build failed)**  
→ Убедитесь, что в репозитории есть файл `requirements.txt` и в нём указаны зависимости. В **Settings** в **Build Command** можно оставить пустым или указать: `pip install -r requirements.txt`.

**Бот не отвечает в Telegram**  
→ Проверьте логи: есть ли «Бот запущен...» и нет ли красных сообщений об ошибке. Убедитесь, что **Start Command** именно `python bot.py`.

**Где взять ADMIN_ID и CHANNEL_ID?**  
→ ADMIN_ID: напишите боту @userinfobot в Telegram — он пришлёт ваш ID. CHANNEL_ID: у канала с ботом-администратором можно узнать через API или боты вроде @getidsbot (ID канала обычно отрицательное, например `-1001234567890`).

## Итог

После выполнения шагов 1–6 бот работает на серверах Railway постоянно. Вы можете закрыть браузер и компьютер — клиенты смогут записываться на маникюр в любое время. При падении процесса Railway автоматически перезапустит бота.

Для обновления бота: измените код, сделайте `git add .`, `git commit -m "..."`, `git push` — Railway сам соберёт и задеплоит новую версию.
