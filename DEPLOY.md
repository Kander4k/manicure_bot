# Как запустить бота 24/7 (чтобы запись работала всегда)

Пока бот запущен только на вашем компьютере, он отвечает только когда вы его включили. Чтобы клиенты могли записываться в любое время, бота нужно запустить **на сервере в интернете**.

Ниже — три варианта: два бесплатных облачных и один на своём сервере (VPS).

---

## 1. Railway (проще всего, есть бесплатный уровень)

1. Зарегистрируйтесь на [railway.app](https://railway.app) (через GitHub).
2. **New Project** → **Deploy from GitHub repo** → выберите репозиторий с этим ботом (если репо ещё нет — создайте на GitHub и залейте проект).
3. В настройках сервиса:
   - **Settings** → **Build**: Command оставьте пустым (или `pip install -r requirements.txt`).
   - **Settings** → **Deploy**: Start Command — `python bot.py`.
   - **Variables** → добавьте переменные из вашего `.env`:
     - `BOT_TOKEN`
     - `ADMIN_ID`
     - `CHANNEL_ID`
     - `CHANNEL_LINK`
     - при необходимости `DATABASE_PATH` (по умолчанию будет создана в корне проекта).
4. Нажмите **Deploy**. После сборки бот будет работать постоянно и перезапускаться при падении.

База данных на Railway при перезапуске может сбрасываться (если не подключить отдельный объём). Для постоянного хранения данных можно позже добавить PostgreSQL и перейти на него или использовать другой хостинг с диском.

---

## 2. Render (бесплатный Background Worker)

1. Зарегистрируйтесь на [render.com](https://render.com).
2. **New** → **Background Worker**.
3. Подключите репозиторий с ботом (GitHub).
4. Укажите:
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`
5. В **Environment** добавьте переменные из `.env` (BOT_TOKEN, ADMIN_ID, CHANNEL_ID, CHANNEL_LINK).
6. Создайте Worker. Бот будет работать в фоне постоянно.

На бесплатном плане Worker может «засыпать» после периода неактивности; для бота это обычно не критично — он снова проснётся при следующем обращении.

---

## 3. Свой сервер (VPS, Linux)

Если есть или планируете арендовать VPS (Hetzner, Timeweb, DigitalOcean и т.п.):

1. Установите Python 3.10+ и склонируйте проект на сервер.
2. Создайте виртуальное окружение, установите зависимости:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```
3. Положите в папку проекта файл `.env` с BOT_TOKEN, ADMIN_ID, CHANNEL_ID, CHANNEL_LINK.
4. Используйте systemd, чтобы бот работал как служба и перезапускался при сбоях:
   - Файл-пример: `deploy/manicure-bot.service`.
   - Скопируйте его в `/etc/systemd/system/`, подставьте своего пользователя и пути, затем:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable manicure-bot
   sudo systemctl start manicure-bot
   ```
   Логи: `journalctl -u manicure-bot -f`.

После этого бот будет работать автоматически и независимо от того, залогинены вы на сервер или нет.

---

## Кратко

| Вариант   | Плюсы                    | Минусы                          |
|----------|---------------------------|----------------------------------|
| Railway  | Быстро, из коробки        | Лимиты бесплатного плана        |
| Render   | Бесплатный worker         | Может «засыпать» на бесплатно   |
| VPS      | Полный контроль, стабильно | Нужно настраивать и платить за сервер |

Для начала удобнее всего попробовать **Railway** или **Render**: залить код в GitHub, подключить репо к сервису, прописать переменные из `.env` — бот начнёт работать автоматически, и запись на маникюр будет доступна людям в любое время.
